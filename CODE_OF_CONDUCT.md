# Code Of Conduct

`gitdex` is built for calm, rigorous collaboration.

## Our standard

Participants in this repository are expected to:

- discuss ideas, code, and tradeoffs without personal attacks
- assume good intent while still being technically precise
- accept review and correction without hostility
- keep issue reports, pull requests, and discussions focused on the work
- respect different levels of experience, background, and language fluency

## Unacceptable behavior

The following behavior is not acceptable in issues, discussions, pull requests, or other project channels:

- harassment, intimidation, or threats
- discriminatory or demeaning language
- publishing private information without permission
- deliberate disruption, spam, or bad-faith escalation
- repeated refusal to follow repository guidance after clear direction

## Maintainer responsibilities

Maintainers may edit, hide, lock, or remove comments, issues, pull requests, and other contributions that do not meet this standard. Maintainers may also temporarily or permanently restrict participation when needed to protect the project.

## Reporting

If you see a conduct issue, open a private security-style report through the contact path in [SECURITY.md](SECURITY.md). Include links, screenshots, or timestamps when possible so the report can be evaluated quickly.

## Scope

This code of conduct applies to repository discussions, issues, pull requests, release notes, and any other official project spaces.

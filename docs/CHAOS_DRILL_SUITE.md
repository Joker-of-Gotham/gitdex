# CHAOS DRILL SUITE

本文件定义 V3 的混沌演练场景，覆盖网络、认证、文件损坏三类高风险故障。

## 演练目标

- 验证系统在异常输入/环境下仍能给出可诊断、可恢复的结果。
- 避免“静默失败”与“死循环重试”。
- 保证关键状态文件损坏后可自动自愈或可快速人工恢复。

## 场景矩阵

- **网络故障**：超时、连接中断、5xx 抖动。
  - 预期：动作结果归类为可重试（`retry`），并给出明确诊断。
- **认证故障**：401/403、scope 不足、未登录。
  - 预期：动作结果归类为人工处理（`manual`），阻断盲目重试。
- **文件损坏**：`.gitdex/maintain/output.txt` 非法 JSON。
  - 预期：自动隔离损坏文件并重建健康文件，流程不中断。

## 自动化测试入口

- `go test ./internal/executor -run ChaosDrill -count=1`
- `go test ./internal/dotgitdex -run Corrupted -count=1`

## 通过标准

- 所有场景测试通过。
- 不产生 panic。
- 日志中可看到明确分类（网络/认证/文件损坏）与恢复建议。

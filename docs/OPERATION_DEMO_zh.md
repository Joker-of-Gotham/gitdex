# gitdex 操作演示

这份文档按一次真实操作路径演示 `gitdex` 的关键交互，目标是让你在几分钟内完成一轮完整验证。

## 适用场景

假设你现在位于一个真实 Git 仓库中，希望完成这些事：

- 看清当前仓库状态
- 让 AI 给出下一步建议
- 查看 Context、Raw、Thinking 和执行结果
- 处理 advisory、命令建议和补参数建议
- 在运行中切换界面语言

## 1. 启动

在仓库根目录执行：

```powershell
go run .\cmd\gitdex
```

或者：

```powershell
.\bin\gitdex.exe
```

## 2. 首次进入先选语言

如果这是这台机器第一次运行 `gitdex`，TUI 会先进入语言选择界面。

操作：

- `up/down` 或 `j/k`：移动
- `Enter`：确认

你应验证：

- 选中文后，界面会立即切到中文
- 之后在主界面按 `L`，还能再次打开语言设置

## 3. 选择模型

如果本地 Ollama 可用，下一步会进入模型选择界面。

流程：

1. 先选主模型
2. 再决定是否选 verifier 模型
3. 如果不需要 verifier，按 `s` 跳过

建议：

- 主模型先用 `qwen2.5:3b`
- 如果你希望校验更稳，再加一个更大的 verifier

## 4. 进入主界面后先看哪里

主界面主要分成三块：

- 左侧：`AI analysis`、`Suggestions`、`Operation Log`
- 右上：`Git Areas`
- 右下：`Observability`

先按一次 `o` 或 `O`，确认右侧 inspector 可以在这些面板之间切换：

- Workflow
- Timeline
- Context
- Memory
- Raw
- Result
- Thinking

## 5. 设置目标或选择工作流

### 自由目标

按 `g`：

- 输入一个目标，例如：`整理当前改动并准备发布`
- 按 `Enter`

这会触发一轮新的分析。

### 选择工作流

按 `f`：

- 选择一个 workflow
- 按 `Enter`

如果前置条件不满足，状态栏会显示 blocked 原因。

## 6. 处理 advisory 建议

如果当前建议是一条 advisory，卡片会明确标注：

```text
(仅查看；不会执行命令；按 y 标记为已查看)
```

对它按 `y` 之后，预期行为是：

1. 不会执行任何命令
2. `Timeline` 里出现 `Viewed advisory: ...`
3. 当前建议会被标记为已查看
4. 下一轮分析不会再重复同一条 advisory

这一段就是专门用来验证此前那个“查看类建议死循环”问题已经修掉。

## 7. 处理可执行建议

如果当前建议是一条真实命令建议：

- 按 `y` 执行
- 看状态栏、`Timeline` 和 `Result`

你应看到：

- `Executing: ...`
- 命令成功或失败的结果
- 最近结果列表更新

如果命令失败，`Result` 会显示失败输出。

## 8. 处理补参数建议

如果建议需要补参数，按 `y` 后会进入输入界面。

操作：

- 输入参数
- `Tab`：切到下一个字段
- `Enter`：确认并执行
- `Esc`：取消

验证点：

- 顶部会显示命令预览
- 参数填完后，执行结果会进入 `Timeline` 和 `Result`

## 9. 依次查看 Observability 面板

建议按这个顺序切一遍：

### Workflow

看：

- 当前阶段
- 当前目标
- 建议总数 / 待处理 / 已完成
- 最近事件

### Timeline

看：

- 状态刷新
- LLM 开始分析
- LLM 输出
- 用户接受 / 跳过 / 查看 advisory
- 命令执行结果

### Context

看：

- 当前模式
- 主模型 / verifier
- 上下文预算
- Prompt 分区
- 最近操作注入
- Prompt / System 摘要

### Memory

看：

- 仓库记忆文件路径
- 偏好
- 仓库模式
- 已解决目标
- 会话目标历史

### Raw

看：

- 原始 LLM 输出
- 清洗后的结构化文本
- 被拒绝的建议

### Result

看：

- 最近一次命令或 advisory 的状态
- 目标
- 输出
- 最近结果列表

### Thinking

如果模型暴露 reasoning 文本，这里会显示；如果没有，会给出明确提示。

## 10. 运行中修改语言

在主界面按 `L`：

- 重新打开语言设置
- 选择新语言
- 按 `Enter`

预期结果：

- UI 会立即切换语言
- 配置写入全局 `gitdex` 配置目录
- 下次启动保持这次选择

## 11. 最小回归检查

如果你要快速确认“产品级发布前这一轮优化”是否都生效，至少检查这些点：

1. 首次启动先出现语言选择
2. 模型选择正常
3. 主界面没有乱码或坏图标
4. advisory 按 `y` 后不会死循环重复
5. `Timeline`、`Context`、`Raw`、`Result` 都有内容
6. `L` 可以重新切换语言
7. `go test ./...` 全通过

## 12. 退出

按 `q` 退出。

# GitDex V3 UI 指南

## 目标

- 信息完整可见：关键文本必须换行显示，不使用省略号截断。
- 焦点可感知：当前焦点区必须有明确视觉高亮。
- 键鼠一致：键盘与鼠标滚动行为语义一致。

## 布局规范

- 顶部：状态栏（模式、执行态、上下文使用率）。
- 中部：左右双区（左 Suggestions，右 Git/Goals/Log 三段）。
- 底部：输入区 + 快捷帮助。

## 交互规范

- `Tab` 循环切换焦点区。
- `PgUp/PgDn` 在当前焦点区分页滚动。
- 鼠标滚轮在悬停区滚动；点击切换焦点。
- `?` 或 `/help` 打开帮助浮层；`Esc` 关闭。

## 文本渲染规范

- 优先 `wrapText`，不使用截断函数。
- 统一处理换行：`\r\n`、`\r` 归一化为 `\n`。
- 多行展示时保留结构前缀（命令、原因、输出、错误）。

## 视觉层级规范

- primary：关键信息（标题、焦点、当前流程）。
- success/warning/danger：状态信号。
- muted：辅助信息（说明、提示、次级细节）。

## 设计评估清单

- 是否存在任意核心字段被省略号截断？
- 所有主面板是否可通过键盘与鼠标滚动？
- 帮助系统是否覆盖核心命令与键位？
- 主题切换后层级是否仍清晰？

# Publishing gitdex To GitHub

This guide is the exact publish checklist for the public repository:

```text
git@github.com:Joker-of-Gotham/gitdex.git
```

It assumes the product is ready for the first stable public tag:

```text
v1.0.0
```

## 1. Align The Module Path

The repository should resolve to:

```text
github.com/Joker-of-Gotham/gitdex
```

If you fork under a different owner, use one of the helper scripts:

PowerShell:

```powershell
.\scripts\set-module-path.ps1 -ModulePath github.com/<your-user-or-org>/gitdex
```

Bash:

```bash
./scripts/set-module-path.sh github.com/<your-user-or-org>/gitdex
```

These scripts update:

- `go.mod`
- Go imports under `cmd/`, `internal/`, and `test/`
- docs, GitHub workflows, and helper scripts

## 2. Clean The Workspace

Do not publish generated or local-only artifacts:

- `bin/`
- `dist/`
- `.gitdex/`
- `.cursor/`
- `.opencode/`

These are already ignored and should stay out of the public history.

## 3. Run The Release Gate

Minimum release gate:

```bash
go vet ./...
go test ./...
go build ./...
```

Recommended additional gate:

```bash
go test -race ./...
```

Windows:

```powershell
.\build.ps1 -Target test
.\build.ps1 -Target build
```

## 4. Build The Release Assets

Generate the full asset set locally:

```bash
./scripts/build.sh v1.0.0 dist
```

On Windows, you can generate the same asset set without Bash:

```powershell
.\build.ps1 -Target assets
```

Expected output:

- `dist/gitdex-windows-amd64.exe`
- `dist/gitdex-windows-arm64.exe`
- `dist/gitdex-linux-amd64`
- `dist/gitdex-linux-arm64`
- `dist/gitdex-macos-amd64`
- `dist/gitdex-macos-arm64`
- `dist/gitdex-source.zip`
- `dist/checksums.txt`

GitHub Actions will produce the same set automatically on tag push.

## 5. Check The README Surface

Before the first public push, verify:

- `README.md` renders the hero without clipping
- the quick-link badge rows are visible
- the repository guide links resolve
- the screenshot assets exist:
  - `docs/assets/readme-hero.svg`
  - `docs/assets/readme-observability.svg`
  - `docs/assets/readme-advisory.svg`

## 6. Apply GitHub Repository Settings

Recommended repository description:

```text
AI-native Git workflow for local repositories with visible context, memory, raw output, and execution flow.
```

Recommended website:

```text
https://github.com/Joker-of-Gotham/gitdex#readme
```

Recommended topics:

- `git`
- `tui`
- `ollama`
- `local-first`
- `observability`
- `developer-tools`
- `terminal-ui`
- `ai-workflow`

These settings are configured in the GitHub web UI after the repository is pushed.

## 7. Push The Repository In Batches

Recommended commit order:

1. repo scaffolding and ignore rules
2. core application source
3. docs, assets, and GitHub community files

After the branch is on GitHub:

```bash
git push -u origin main
```

## 8. Create And Push The Release Tag

```bash
git tag v1.0.0
git push origin v1.0.0
```

The `Release` workflow will then:

- run `go vet`
- run `go test -race ./...`
- build all six binaries
- generate `gitdex-source.zip`
- generate `checksums.txt`
- publish the GitHub release with attached assets

## 9. Release Title And Notes

Recommended release title:

```text
gitdex v1.0.0
```

Reference materials:

- [../.github/RELEASE_TEMPLATE.md](../.github/RELEASE_TEMPLATE.md)
- `dist/release-notes.md` generated by `./scripts/build.sh`

## 10. Final Public Checklist

- `README.md` is polished and renders cleanly
- `docs/README_zh.md` and other Chinese docs are readable and not mojibake
- `CODE_OF_CONDUCT.md`, `CONTRIBUTING.md`, and `SECURITY.md` exist
- `.github/workflows/ci.yml`, `.github/workflows/release.yml`, and `.github/workflows/codeql.yml` are active
- the repository uses `github.com/Joker-of-Gotham/gitdex`
- legacy compatibility notes for `.gitmanualrc` and `GITMANUAL_*` are still documented

# Story 2.1: Compile Structured Plans from Commands and Chat

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a maintainer,
I want Gitdex to compile my command or natural-language request into a structured plan before any governed write occurs,
so that I can review scope, risk, and policy outcomes before execution begins.

## Acceptance Criteria

1. **Given** a write-capable request expressed through command or chat **When** Gitdex prepares the request for execution **Then** it produces a structured plan showing target scope, intended steps, risk level, and policy verdict.

2. **And** the plan is stored with a stable identifier and traceable back to the originating request.

3. **And** blocked or escalated outcomes are explained in operator-readable language rather than hidden behind generic errors.

4. **And** missing parameters are collected inline while high-risk parameter changes require an explicit second confirmation before the plan can advance.

- **Task 2:** DefaultCompiler produces structured plans from Intent. Risk assessment: production ? critical, main/master ? high, delete/force_push ? high, default ? low. Steps derived from action type.
- **Task 3:** DefaultPolicyEngine evaluates plans: low ? allowed, medium ? allowed/escalated, high ? escalated (requires repository_admin), critical ? blocked. Risk factors collected for protected branch, production env, irreversible steps. Explanation is operator-readable natural language.
- **Task 4:** MemoryPlanStore with Save/Get/GetByTaskID/List/UpdateStatus. Copy isolation on save/read.
- **Task 5:** `gitdex plan` command group: `compile <goal>`, `show <plan_id>`, `list`. Text/JSON/YAML output. Blocked/escalated results show PolicyVerdict with explanation.
- **Task 6:** 6 test files, 33 packages pass. Tests cover: model serialization, compiler logic, policy evaluation, store CRUD, command registration, contract conformance.

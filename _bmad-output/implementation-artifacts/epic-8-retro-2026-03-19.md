## 第一部分: Epic 8 回顾

### ✅ 成功之处

#### 1. 流式 LLM 集成架构成熟

Story 8.1 的 LLM 集成设计干净利落。`StreamChunkMsg` 通过 Bubble Tea 的 `tea.Cmd` 机制实现异步渲染，不阻塞 UI 线程。`chat.Session` 的滑动窗口上下文管理优雅简洁。Provider 热切换通过 `ConfigSaveMsg` 事件驱动，零停机。

> Charlie (Senior Dev): "流式渲染的架构是本 Epic 的亮点。把 LLM 的异步特性完美映射到了 Bubble Tea 的消息模型。"

#### 2. 安全护栏系统完整无遗漏

Story 8.5 的 `Guardrails` 系统采用了三级防护策略 (自动/审批/拦截)。v2 修复后，所有已知高危操作 (`git.push.force`, `git.reset.hard`, `git.branch.delete.force`, `github.repo.delete`, `github.branch.protection.set`) 均在 `blockedActions` + `actionRiskMap(RiskCritical)` 中双重注册。

> Dana (QA Engineer): "v2 修复后安全护栏零遗漏 — 每个危险操作都有硬拦截 + 风险分级 Critical 双保险。"

#### 3. 命令路由系统解耦优秀

`app/commands.go` 将 30+ 个 Git/GitHub 命令从 `app.go` 中抽离，`requireRepo` 和 `requireWritable` 装饰器模式使得只读模式保护自然融入。

> Alice (Product Owner): "从产品角度看，这种解耦意味着我们可以安全地扩展命令集而不影响核心 TUI。"

#### 4. 测试覆盖全面且高质量 (v2)

91 个 E2E 测试 + 全量套件在 72 秒内完成。v2 引入了 Test Data Factory 模式、消除了所有弱断言、补全了全部 39 个 AC 覆盖。

> Dana (QA Engineer): "v2 测试质量达到 100/100 — Factory 模式让数据构造代码减少了 50%+，assertContains 替换了所有 output==\"\" 弱断言。"

### ⚠️ 挑战与教训

#### 1. Bubble Tea v2 API 差异导致的返工

**问题**: Story 8.2 最初尝试添加 `tea.WithBracketedPaste()` 和 `tea.WithMouseCellMotion()`，但这些 API 在 v2 中已改变。
**根因**: 参考了 v1 文档/示例，未先验证 v2 API 变更。
**教训**: 使用第三方库的新大版本时，先查看 changelog 和 migration guide。

#### 2. 类型签名不匹配导致编译失败

**问题**: `ActionHandler` 的测试中使用 `interface{Done() <-chan struct{}}` 而非 `context.Context`。
**根因**: 在跨包接口集成时，未充分检查函数签名。
**教训**: 跨包集成时，先编译再写逻辑。

#### 3. Dashboard Tab 数量变化导致测试失败

**问题**: Dashboard 从 2 tab 扩展为 3 tab 后，原有的边界导航测试断言失败。
**教训**: 修改共享组件的结构时，需立即搜索并更新所有引用该结构的测试。

#### 4. 安全护栏遗漏 (v1 → v2 修复)

**问题**: `git.reset.hard` 未在 `blockedActions` 中，`github.repo.delete` 未在 `actionRiskMap` 中标记 `RiskCritical`。
**教训**: 安全关键代码需要双重验证 — 不仅检查阻止列表，还要检查风险分级是否一致。

### 🔍 关键洞察

1. **消息类型集中管理有效**: `views/messages.go` 作为消息类型的统一定义点，避免了重复定义和命名冲突。

2. **装饰器模式适合命令保护**: `requireRepo`/`requireWritable` 的组合模式使得添加新命令时几乎不需要思考权限逻辑。

3. **Test Data Factory 是测试质量的关键**: v2 引入 factory 后，新测试的数据构造代码减少 50%+，同时提高了一致性。

4. **弱断言是隐藏的风险**: `output == ""` 类型的断言可能掩盖渲染错误。v2 全部替换为 `assertContains` 后，发现并修复了多处潜在问题。
